## 执行流程

基于 D01/D02 宽表 10% 抽样数据，完成了从样本检查、特征筛选、模型训练到评估对比的完整建模链路。在首轮模型与 gcard_v6 持平后，执行了 Null Importance 阈值放宽（1.0→0.5, 特征 360→679）+ 50 轮 Optuna 超参搜索。

### 关键指标

| 指标 | 首轮 | Optuna优化 | 变化 |
|------|------|-----------|------|
| 最终入模特征数 | 360 | 679 | +319 |
| DEV AUC | 0.9461 | 0.9535 | +0.0074 |
| DEV-OOS AUC | 0.9334 | 0.9338 | +0.0004 |
| OOT AUC | 0.9330 | 0.9358 | +0.0028 |
| DEV-OOS KS | 0.7212 | 0.7220 | +0.0008 |
| AUC Gap | 0.0127 | 0.0197 | +0.0070 |
| Score PSI (max) | 0.0183 | 0.0578 | +0.0395 |

### vs 历史冠军 (DEV-OOS)

| 冠军 | 首轮AUC差 | Optuna AUC差 | KS差 |
|------|-----------|-------------|------|

| gcard_v5 | +0.0098 | +0.0102 | +0.0216 |
| gcard_v6 | -0.0002 | **+0.0002** | **+0.0024** |

### 结论

1. Optuna 超参搜索 + Null Importance 放宽带来的 AUC 提升仅 0.0004 (DEV-OOS)，处于统计噪声范围。
2. 模型显著优于 gcard_v2/v4/v5，但与 gcard_v6 基本持平。
3. AUC Gap 从 0.0127 恶化至 0.0197，更接近过拟合阈值。
4. Score PSI 从 0.0183 升至 0.0578，稳定性有所下降。
5. **不推荐替换 gcard_v6**：提升不显著 + B2 客群退化 + 过拟合风险升高。

### 下一步建议

1. **扩大样本量**（Tier 1）：使用全量而非 10% 抽样进行训练，是当前最可能带来实质提升的手段。
2. **增量特征工程**：针对 B2 流失户和 E2 次新客群挖掘新特征。
3. **探索先进建模方法**：如 TabNet、深度交叉网络等。
4. **多模型 Ensemble**：将 gcard_v6 与本轮模型进行集成。
