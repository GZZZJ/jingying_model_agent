# 复借 G 卡主模型重跑 — 高管摘要

**Run**: `2026-06-26-gcard-main-lgbm-rerun`　**日期**: 2026-06-26　**负责人**: 辜子骏
**数据口径**: 本地 feather（复借 G 卡 D01/D02 宽表，489,743 行 × 2,857 列，2025-06 ~ 2026-01）

## 1. 目标

基于最新复借 G 卡宽表口径，从 0 重跑一版全客群复借意愿主模型（LightGBM），
形成可审计的新 run 产物，并与历史 G 卡模型做 champion/challenger 对比。不复用旧 run 完成状态。

## 2. 核心结论

- **新模型（main_lgbm）已达到生产可用水平**：与最强历史版本 G卡V6 在时间外样本（OOT）上基本持平。
- **OOT 全客群**：AUC 0.934 / KS 0.733（G卡V6 为 0.936 / 0.734，差距 −0.002 AUC，统计意义上持平）。
- **稳定优于其他历史版本**：相对 gcard_v2/v4/v5，OOT AUC 提升 +0.006 ~ +0.011。
- **泛化稳健**：DEV→OOT AUC 仅由 0.945 降至 0.934（−0.011），无过拟合衰减；
  auc_gap 0.013 < 0.02 阈值。
- **月度稳定**：分数 PSI 月度最大 0.052（< 0.1 稳定线）；8 个月 AUC 区间 0.928 ~ 0.948，无塌陷。
- **排序单调**：十分箱 0 倒挂。

## 5. 下一步建议

1. 将 main_lgbm 作为 G卡V6 的并列 challenger 上线灰度，重点观察 OOT 之外的未来月份 PSI 与分客群（B2/E2E3）稳定性。
2. 持续监控月度正样本率下行对校准的影响，必要时做 score 校准。
3. 评估是否引入 G卡V6 之外的增量特征域以突破当前提升瓶颈。
