# 复借G卡名词口径

更新时间：2026-05-30

本文记录复借G卡项目当前已确认或待确认的业务名词口径，供样本切片、报告解释和后续同事确认时引用。

## 客群口径

| 名词 | 字段取值 | 含义 | 当前使用方式 |
|---|---|---|---|
| 流失户 B2 | `blue_customer_flag = 'B2'` | 结清30天+；2024.08 开始交易侧口径变更为“主营结清30+，且2个月无发起”；额度侧流失话口径未变；2025-07-02 变更为 180 天未发起。 | 作为流失户客群切片和分客群建模候选。 |
| 次新户 E2 | `blue_customer_flag = 'E2'` | 首借后30-100天；后续 E1 也会在贷中经营，但是口径还没完全变化。 | 与 E3 合并为老户次新客群，必要时单独切片。 |
| 老户 E3 | `blue_customer_flag = 'E3'` | 首借后100天+。 | 与 E2 合并为老户次新客群，必要时单独切片。 |

复借G卡历史报告中的两个主要分客群可先按以下映射处理：

- 老户次新：`blue_customer_flag in ['E2', 'E3']`
- 流失户：`blue_customer_flag == 'B2'`

## 相关业务名词

| 名词 | 含义 |
|---|---|
| B卡 beta版 | 相较 B 卡，多资信数据源。以 V16 为例，会多腾讯、蚂蚁链和百度的联合建模分。 |
| DCP | 公司一个类似信用卡的产品，还享花。主要是1期订单，每个人每个月有固定还款日，定价都是24%。 |
| 风险切口径 | 锚定一个时间切片看数据，例如按不同交易月观测风险。 |
| 风险V口径 | 按首借月看不同账龄变化，锚定的是固定的人。 |
| Δcps | Δ营销成本 / Δ交易金额。先要求为正，然后越小越好。 |
| takerate | Δ损益 / Δ交易金额。损益口径：利息收入 - 资金成本 - 运营成本 - 风险成本 - 提前结清成本；运营成本 = 资信成本 + 第三方支付成本 + 催收成本 + 产品成本 + 客经成本。 |
| roi | Δ损益 / Δ营销成本。 |
| CPA | 成本 / 授信数。 |

## 风险表现口径

当前样本表已有字段：

- `prc_amt_xz_30d_3m`
- `ovd_amt_xz_30d_3m`

这两个字段可能可用于金额风险观察，但是否与历史报告中的 MOB1/MOB3 人头逾期率、金额逾期率同口径仍未确认。

待同事确认：

1. MOB1/MOB3 的观察起点、账龄定义和表现窗口。
2. 人头逾期率的分子、分母和去重主键。
3. 金额逾期率的本金口径、逾期金额口径和是否剔除特殊状态。
4. `prc_amt_xz_30d_3m`、`ovd_amt_xz_30d_3m` 是否能直接复刻历史报告风险表。

## 特征规模约束

后续特征会来自其他表，不在当前基础样本表中完成筛选和建模。

已知约束：

- 候选特征可能超过 15,000 个。
- 样本量为几百万级。
- 特征筛选必须支持随机抽样。
- 特征筛选必须支持分批处理特征。
- 特征筛选结果要保留每步剔除原因、剩余变量清单、变量重要性和可回溯配置。
