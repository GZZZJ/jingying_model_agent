# 复借G卡新训练版本后续训练评估流程计划

生成日期：2026-06-08

本文档用于指导在另一套环境中拉取本项目后，继续完成新一版复借G卡模型训练、评估和 Excel 报告产出。当前目标不是复刻历史 V6 的训练过程，也不是先做分客群建模，而是基于一批新的训练数据、新的特征集合、新的时间窗口，训练一个当前版本的主模型，并按历史 `复借G卡模型文档.xlsx` 的报告结构输出同格式评估 Excel。

## 1. 本轮目标和边界

### 1.1 本轮目标

本轮要完成：

1. 使用新数据窗口和新特征集合训练一个复借G卡主模型。
2. 记录完整训练输入：样本来源、时间窗口、切分规则、标签、特征清单、训练参数。
3. 在 DEV/OOT/OOS 等可用切分上评估新模型。
4. 将新模型分与历史版本分 `gcard_v2`、`gcard_v4`、`gcard_v5`、`gcard_v6` 做对比。
5. 按历史 Excel 的结构输出报告，最终产物是 `reports/model_report.xlsx`。

### 1.2 本轮不做的事

本轮暂不做：

- 不训练 `e2e3_lgbm`、`b2_lgbm` 等分客群模型。
- 不把老户次新、流失户作为单独训练实验。
- 不把历史文档中的旧时间窗口当作本轮训练窗口。
- 不强行复刻历史变量数 `380/339/252`。

分客群仍然要评估，但只作为效果切片：

- 全客群
- 老户次新：`blue_customer_flag in ('E2','E3')`
- 老户：`blue_customer_flag == 'E3'`
- 次新：`blue_customer_flag == 'E2'`
- 流失户：`blue_customer_flag == 'B2'`

## 4. 训练方案

### 4.1 主训练实验

本轮只训练一个主模型：

| 实验名 | 说明 |
| --- | --- |
| `main_lgbm` | 基于新数据、新特征、新时间窗口训练的复借G卡主模型 |

默认训练样本：

- Train：`final_flag == 'DEV'`
- Valid/OOT：`final_flag == 'OOT'`
- OOS：如果存在，则只用于最终外推评估，不参与训练和 early stopping

客群字段只用于评估切片，不用于拆分训练。

### 4.2 可选训练方式

因为本轮可能会尝试新的模型训练方式，训练脚本应支持但不强制开启：

| 方式 | 默认状态 | 用途 |
| --- | --- | --- |
| LightGBM binary | 默认 | 主训练方式 |
| XGBoost binary | 可选 | 与 LightGBM 对比 |
| Logistic/Scorecard | 可选 | 可解释性 baseline |
| topN 特征敏感性 | 可选 | 比较 top 50/100/200/全部特征 |
| 参数网格或 Optuna | 可选 | 后续调参 |

正式报告只推荐一个主模型版本。如果做了可选实验，结果放到附录，不改变主报告结构。

### 4.3 训练预处理要求

训练脚本必须做到：

- 所有特征按训练清单取列。
- 缺失特征记录到 `feature_availability.csv`，不能静默忽略。
- 特征统一转数值，无法转数值置为缺失。
- `-999`、`-998`、正负无穷置为缺失。
- 缺失填充策略只能从训练集学习，并保存到 `preprocessing.json`。
- 删除训练集中非空率过低或唯一值小于等于 1 的特征，并保存删除原因。
- 保存最终实际入模特征，不允许只保存候选特征。

## 6. 评估方案

### 6.1 评估原则

评估脚本只评估一个新模型分 `model_score`，并与历史版本分对比。

历史版本分使用规则：

- 全客群：对比所有可用历史分。
- 老户次新：重点对比 `gcard_v2`、`gcard_v4`。
- 流失户：重点对比 `gcard_v2`、`gcard_v5`。
- `gcard_v6` 如果在样本中存在，用作上一代整体版本参考。

分客群只做切片评估，不输出“分客群模型”指标。

### 6.2 必须输出的评估表

评估产物目录：

```text
runs/model_eval/
  overall_metrics.csv
  monthly_metrics.csv
  segment_metrics.csv
  decile_lift_all.csv
  decile_lift_e2e3.csv
  decile_lift_b2.csv
  intent_zc_distribution.csv
  intent_zc_ftr_rate.csv
  intent_zc_amount_risk.csv
  intent_zc_headcount_risk.csv
  score_psi_by_month.csv
  benchmark_uplift.csv
  evaluation_summary.json
```

#### 整体效果表

维度：

- `final_flag`
- 可用时补充 `mdl_month`

分数：

- `model_score`
- `gcard_v2`
- `gcard_v4`
- `gcard_v5`
- `gcard_v6`

指标：

- 样本数
- 正样本数
- 正样本率
- AUC
- KS

#### 月度效果表

维度：

- `mdl_month` 或 `mdl_dte`
- `final_flag`

指标：

- AUC
- KS
- 样本数
- 正样本率

#### 客群切片效果表

切片：

- 全客群
- 老户次新：`E2 + E3`
- 老户：`E3`
- 次新：`E2`
- 流失户：`B2`

指标：

- AUC
- KS
- 样本数
- 正样本率
- 相对历史版本 KS uplift

#### 十分位 lift/sloping 表

分箱：

- 每个分数在目标评估样本内等频 10 箱。
- 分数方向必须统一：高分代表更高 30 天发起概率。

输出：

- 分组
- 占比
- 箱内发起率
- 累计发起率
- 累计 lift
- 剩余发起率
- 剩余 lift

切片：

- 全客群
- 老户次新
- 流失户

#### 意愿评级 x 资产评级交叉表

意愿评级：

- 按模型分三等频切成低/中/高意愿。

交叉维度：

- `zc_level`

输出：

- 占比
- 30 天发起率
- 低意愿评级下发起率与资产评级线性相关性

#### 风险观察表

当前可用字段：

- `prc_amt_xz_30d_3m`
- `ovd_amt_xz_30d_3m`

先输出：

- 金额逾期率：`sum(ovd_amt_xz_30d_3m) / sum(prc_amt_xz_30d_3m)`
- 人头风险率：如果没有明确 MOB 口径，暂按 `ovd_amt_xz_30d_3m > 0` 做观察口径

注意：

- MOB1/MOB3 人头逾期率、金额逾期率历史计算逻辑仍待确认。
- 未确认前，报告中只能写“风险观察”，不能写“历史口径复刻”。

#### 稳定性表

输出：

- 模型分 10 箱跨月占比
- 模型分 10 箱跨月发起率
- PSI by month
- 如果最新月份尾部偏移明显，在报告中单独提示。

## 7. Excel 报告格式

目标报告：

```text
reports/model_report.xlsx
```

必须尽量保持历史 `doc/复借G卡模型文档.xlsx` 的阅读结构，建议 sheet：

| Sheet | 内容 |
| --- | --- |
| `Summary` | 模型描述、样本定义、变量筛选过程、核心效果结论、重点 benchmark |
| `模型效果` | 整体 DEV/OOT/OOS AUC/KS、月度效果、历史版本对比 |
| `分客群效果` | 老户次新、老户、次新、流失户切片效果；注意不是分客群训练模型 |
| `Lift` | 十分位累计发起率、累计 lift、剩余发起率、剩余 lift |
| `意愿资产交叉` | 高中低意愿 x `zc_level` 的占比和发起率 |
| `风险观察` | 金额逾期率、人头风险率、MOB 口径待确认说明 |
| `稳定性` | 分箱跨月分布、跨月 PSI |
| `重要变量` | 入模变量、重要性、变量说明、特征来源 |
| `Source Data` | 所有用于报告的明细表，便于审计 |

`Summary` 页必须写清：

- 本轮是新数据、新特征、新时间窗口训练版本。
- 本轮暂不做分客群建模。
- 分客群结果只是切片评估。
- 训练样本窗口、验证/OOT/OOS 窗口。
- 最终入模特征数。
- 主模型算法和参数。
- 新模型相对历史版本的核心 AUC/KS uplift。
- 风险观察的口径限制。

## 8. 需要补齐的脚本

### 8.1 `scripts/03_train.py`

目标：正式训练单一主模型。

建议参数：

```bash
python3 scripts/03_train.py \
  --input-feather data/local/modeling_sample.feather \
  --feature-list runs/modeling_feature_set/feature_list.txt \
  --output-dir runs/model_train/main_lgbm \
  --score-output runs/model_scores/scores_all_splits.feather \
  --config configs/train.yaml
```

脚本职责：

### 8.2 `scripts/04_evaluate.py`

目标：基于统一预测表输出所有评估 CSV/JSON。

建议参数：

```bash
python3 scripts/04_evaluate.py \
  --scores-feather runs/model_scores/scores_all_splits.feather \
  --output-dir runs/model_eval \
  --config configs/evaluate.yaml
```

脚本职责：

- 计算整体、月度、客群切片 AUC/KS。
- 计算历史版本 benchmark 和 uplift。
- 计算 decile lift。
- 计算意愿评级 x 资产评级交叉。
- 计算风险观察表。
- 计算稳定性和 PSI。
- 输出 `evaluation_summary.json`。

### 8.3 `scripts/05_report.py`

目标：生成和历史文档格式一致的 Excel 报告。

建议参数：

```bash
python3 scripts/05_report.py \
  --eval-dir runs/model_eval \
  --train-dir runs/model_train/main_lgbm \
  --input-dir runs/modeling_input \
  --feature-dir runs/modeling_feature_set \
  --output reports/model_report.xlsx
```

脚本职责：

- 读取训练、评估、特征和样本快照。
- 按第 7 节 sheet 结构生成 Excel。
- `Summary` 页写核心结论。
- `Source Data` 页保留所有评估明细。
- 对缺失的 OOS 或 MOB 口径显示明确说明。

1. 当前 `03_train.py`、`04_evaluate.py`、`05_report.py` 仍需补齐后才能一键执行。
